WebSocket là gì? Tại sao Chat App lại cần nó?

HTTP vs WebSocket HTTP: Mỗi lần muốn lấy dữ liệu mới, Client phải “hỏi” Server (Request) rồi Server mới trả lời (Response). Giống như bạn nhắn tin hỏi người yêu: “Em ăn cơm chưa?” rồi chờ trả lời. WebSocket: Tạo một đường ống kết nối 2 chiều liên tục. Server có thể tự động đẩy tin nhắn về Client bất cứ lúc nào mà không cần hỏi. Giống như đang gọi điện thoại nói chuyện trực tiếp. Code Demo WebSocket Client // 1. Tạo kết nối đến Server const socket = new WebSocket('wss://echo.websocket.org'); // 2. Lắng nghe khi kết nối thành công socket.onopen = function(event) { console.log("Đã kết nối!"); socket.send("Xin chào Server!"); // Gửi tin nhắn }; // 3. Nhận tin nhắn từ Server gửi về socket.onmessage = function(event) { console.log("Tin nhắn từ Server:", event.data); };

Hướng dẫn Fetch API: Cách lấy dữ liệu từ Server chuẩn hiện đại

Fetch API là gì? fetch() là một hàm có sẵn trong mọi trình duyệt hiện đại, dùng để gửi các yêu cầu HTTP (GET, POST, PUT…) đến máy chủ mà không cần thư viện ngoài như Axios hay jQuery. 1. Gửi yêu cầu GET (Lấy dữ liệu) fetch('[https://jsonplaceholder.typicode.com/posts/1](https://jsonplaceholder.typicode.com/posts/1)') .then(response => response.json()) .then(json => console.log(json)); 2. Gửi yêu cầu POST (Tạo dữ liệu mới) fetch('[https://jsonplaceholder.typicode.com/posts](https://jsonplaceholder.typicode.com/posts)', { method: 'POST', headers: { 'Content-Type': 'application/json', }, body: JSON.stringify({ title: 'Bài viết mới', body: 'Nội dung bài viết...', userId: 1, }), }) .then(res => res.json()) .then(data => console.log("Thành công:", data)); Làm chủ Fetch API là kỹ năng bắt buộc của mọi Frontend Developer!
